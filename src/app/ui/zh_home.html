概观

使用 WordPress 或其他内容管理系统 (CMS) 创建具有专业外观的网站很容易，但如果您希望您的网站看起来与普通主题略有不同，则必须自定义设计。 这可能会变得复杂。 （我发现直接修改 HTML 代码比修改 WordPress 主题更容易。）

如果您正在创建电子商务网站，或需要用户登录，那么 WordPress 是一个不错的选择，因为它包含您需要的安全性。

但是，如果您需要与用户进行的最复杂的交互是联系表单，并且您希望使用共享模板、干净的 URL、自动语言选择以及站点设计中的更多灵活性来构建页面，那么此处演示的设置可能是一个很好的解决方案 .

我在详细信息部分提供了有关如何设置站点的说明，并且我的所有文件都在GitHub上。

Bootstrap

Bootstrap 框架提供了一个响应式网格，可以在不同的显示尺寸之间进行调整，并且由于它的流行，互联网上有大量的代码片段可用。

2021 年，Bootstrap 发布了该框架的第 5 版，它利用了 CSS 属性，例如变量 (--sc-navbar-bg: #0024a7;)、计算函数 (width: calc(100% - 80px);)，以及许多 伪类（:is(section, article, aside, nav)）。 CSS 变量对于支持 Bootstrap 5.3 中的暗模式特别有用。

在切换到使用较新的 CSS 属性时，Bootstrap 放弃了对不支持这些属性的 Microsoft Internet Explorer (IE) 的支持。 对 IE 的支持变得不那么重要了，因为在 2022 年，微软终止了对 IE 的支持，并在 2023 年第一季度开始在某些版本的 Windows 10 上永久禁用 Internet Explorer。

永久禁用 Internet Explorer

在全球范围内，仍有超过 8000 万个运行 Windows 7 或 Windows XP 的桌面系统可能正在使用 Internet Explorer。 对于需要支持这些用户的站点，Bootstrap 4 仍然可用。

Fat-Free

虽然Fat-Free不是最流行的PHP框架（可能是Laravel） ，但它确实适用于共享主机方案（Laravel不能，至少不容易），而Fat-Free被认为具有快速的学习曲线

我正在使用的主要无脂组件是：

  • 路由引擎 – 将用户的请求路由到适当的控制功能。
  • 模板引擎 – 将HTML模板与存储在Fat-Free的“hive”中的数据相结合，内存中的数组以键/值对的形式保存变量，以形成网页。
  • 多语言支持 – Fat-Free通过使用键/值对的字典查找功能支持多种语言。 每种语言的字典文件都位于dict目录中，Fat-Free根据模板中的标记和LANGUAGE设置从字典文件中选择所需的语言字符串。
    • 我对所有页面中相同的内容（例如，滚动到顶部按钮）或对所有语言都有共同且文本有限的页面（例如，联系页面）使用无Fat语言支持。
  • 语言检测 Fat-Free从用户的GET请求中捕获Accept-Language字符串，用于在用户请求根目录时选择适当的语言子目录 。
