Bootstrap 5 및 Fat-Free Framework에 대한 경험을 얻기 위해이 사이트를 만들었습니다.

Bootstrap 5

부트 스트랩은 반응 형 웹 사이트를 구축하기위한 널리 사용되는 프론트 엔드 프레임 워크입니다.

Fat-Free Framework

Fat-Free는 상위 10 개 PHP 프레임 워크에서 선정되었습니다.

개요

WordPress 또는 기타 콘텐츠 관리 시스템(CMS)을 사용하여 전문적인 느낌의 웹사이트를 만드는 것은 쉽지만 스톡 테마와 조금 다르게 사이트를 보이게 하려면 디자인을 사용자 지정해야 합니다. 복잡해질 수 있습니다. (WordPress 테마를 수정하는 것보다 직선 HTML 코드를 수정하는 것이 더 쉽다는 것을 알았습니다.)

전자 상거래 사이트를 만들거나 사용자가 로그인해야 하는 경우 WordPress는 원하는 보안 기능을 포함하므로 좋은 옵션입니다.

그러나 사용자와의 가장 복잡한 상호 작용이 문의 양식이고 공유 템플릿, 깔끔한 URL, 자동 언어 선택 및 사이트 디자인의 더 많은 유연성으로 페이지를 구축하려는 경우 여기에 설명된 설정이 좋은 솔루션이 될 수 있습니다.

세부 섹션에서 사이트 설정 방법에 대한 설명을 제공했으며 모든 파일은 GitHub에 있습니다.

Bootstrap

Bootstrap는 다양한 디스플레이 크기에 걸쳐 조정되는 반응형 그리드를 제공하며, 그 인기로 인해 인터넷에서 사용할 수 있는 많은 코드 스니펫이 있습니다.

2021년 Bootstrap은 변수(--sc-navbar-bg: #0024a7;), 계산 함수(width: calc(100% - 80px);) 및 많은 CSS 속성을 활용하는 프레임워크 버전 5를 출시했습니다. 가상 클래스(:is(section, article, aside, nav)). CSS 변수는 Bootstrap 5.3에서 다크 모드를 지원하는 데 특히 유용합니다.

최신 CSS 속성을 사용하도록 전환하면서 Bootstrap은 속성을 지원하지 않는 Microsoft Internet Explorer(IE)에 대한 지원을 중단했습니다. IE에 대한 지원은 2022년에 Microsoft가 IE에 대한 지원을 종료하고 2023년 1분기에 일부 Windows 10 버전에서 Internet Explorer를 영구적으로 비활성화하기 시작했기 때문에 덜 중요해지고 있습니다.

전 세계적으로 Internet Explorer를 사용하고 있을 수 있는 Windows 7 또는 Windows XP를 실행하는 데스크톱 시스템이 여전히 8천만 대 이상 있습니다. 이러한 사용자를 지원해야 하는 사이트의 경우 Bootstrap 4를 계속 사용할 수 있습니다.

Fat-Free

Fat-Free 가 가장 인기있는 PHP 프레임 워크 (Laravel 는 아니지만), 공유 호스팅 계획 (Laravel doesn’t 은 쉽지 않음 ) 에서 작동 하며 Fat-Free는 빠른 학습 곡선으로 간주됩니다.

내가 사용하고있는 주요 Fat-Free 구성 요소는 다음과 같습니다.

  • 라우팅 엔진 – 사용자의 요청을 적절한 제어 기능으로 라우팅 합니다.
  • 템플릿 엔진 – HTML 템플릿을 Fat-Free의 "hive"에 저장된 데이터와 결합합니다. 변수는 키 / 값 쌍 형태의 변수를 메모리에 저장하여 웹 페이지를 구성합니다.
  • 다국어 지원 – Fat-Free는 키 / 값 쌍을 사용하여 사전 조회 기능을 통해 여러 언어를 지원합니다. 각 언어의 사전 파일은 dict 디렉토리에 있으며 Fat-Free는 템플릿의 토큰 및 LANGUAGE 설정에 따라 사전 파일에서 원하는 언어 문자열을 선택합니다.
    • 모든 페이지에서 동일한 내용 (예 : Scroll-to-top 버튼) 또는 모든 언어에서 공통적이고 제한된 텍스트 (예 : 연락처 페이지)가있는 페이지에 대해서는 무 지방 언어 지원을 사용합니다.
  • 언어 감지 – Fat-Free는 사용자의 GET 요청에서 Accept-Language 문자열을 캡처합니다.이 문자열은 사용자가 루트 디렉토리를 요청할 때 적절한 언어 하위 디렉토리 를 선택하는 데 사용됩니다.
